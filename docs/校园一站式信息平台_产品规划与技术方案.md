# 校园一站式信息平台 — 产品规划与技术方案

> 课程设计项目 · 单体架构 · 简洁实用

---

## 一、项目概述

### 1.1 项目背景

随着高校信息化程度不断提升，学生在日常学习和生活中需要访问多个独立系统（教务系统、通知公告、校园活动、图书馆等），操作分散、体验割裂。本项目旨在构建一个**校园一站式信息平台**，将常用功能整合在同一个应用中，提升学生的使用体验。

### 1.2 项目目标

- 整合校园常用信息入口，减少学生在多系统间切换的成本
- 提供简洁、易用的 Web 界面
- 作为课程设计，代码结构清晰、易于理解和维护

### 1.3 目标用户

| 用户角色 | 说明 |
|---------|------|
| 学生 | 主要使用者，查看通知、活动等信息 |
| 教师 | 发布通知、查看班级信息 |
| 管理员 | 后台管理用户、内容审核 |

## 二、功能规划

### 2.1 核心功能模块

本项目保持功能简洁，聚焦以下三个核心模块：

#### 模块一：用户中心
- 注册 / 登录 / 登出
- 个人信息查看与修改
- 密码修改
- 角色区分（学生 / 教师 / 管理员）

#### 模块二：通知公告
- 公告列表展示（分页）
- 公告详情查看
- 管理员发布 / 编辑 / 删除公告
- 按类别筛选（学校通知、院系通知、活动信息）

#### 模块三：校园活动
- 活动列表展示
- 活动详情查看
- 学生报名参加活动
- 管理员 / 教师发布活动

### 2.2 功能优先级

| 模块 | 优先级 | 说明 |
|------|--------|------|
| 用户中心 | P0（必做） | 基础支撑，所有功能依赖登录 |
| 通知公告 | P0（必做） | 核心信息展示功能 |
| 校园活动 | P2（扩展） | 有余力时完成 |

## 三、技术方案

### 3.1 技术选型

本项目为课程设计，采用**单体架构**，技术栈主流且易于上手。

## 四、数据库设计

### 4.1 用户表 `t_user`

```sql
CREATE TABLE t_user (
    id          BIGINT PRIMARY KEY AUTO_INCREMENT,
    username    VARCHAR(50)  NOT NULL UNIQUE COMMENT '用户名',
    password    VARCHAR(255) NOT NULL COMMENT '加密密码',
    real_name   VARCHAR(50)  COMMENT '真实姓名',
    role        TINYINT      NOT NULL DEFAULT 0 COMMENT '0学生 1教师 2管理员',
    email       VARCHAR(100) COMMENT '邮箱',
    created_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### 4.2 通知公告表 `t_notice`

```sql
CREATE TABLE t_notice (
    id          BIGINT PRIMARY KEY AUTO_INCREMENT,
    title       VARCHAR(200) NOT NULL COMMENT '标题',
    content     TEXT         NOT NULL COMMENT '内容',
    category    TINYINT      NOT NULL DEFAULT 0 COMMENT '0学校 1院系 2活动',
    author_id   BIGINT       NOT NULL COMMENT '发布人',
    is_top      TINYINT      NOT NULL DEFAULT 0 COMMENT '是否置顶',
    created_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### 4.3 活动表 `t_activity`

```sql
CREATE TABLE t_activity (
    id           BIGINT PRIMARY KEY AUTO_INCREMENT,
    title        VARCHAR(200) NOT NULL COMMENT '活动名称',
    description  TEXT         COMMENT '活动描述',
    location     VARCHAR(100) COMMENT '活动地点',
    start_time   DATETIME     NOT NULL COMMENT '开始时间',
    end_time     DATETIME     NOT NULL COMMENT '结束时间',
    max_count    INT          DEFAULT 0 COMMENT '最大报名人数，0不限制',
    organizer_id BIGINT       NOT NULL COMMENT '组织者ID',
    created_at   DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE t_activity_signup (
    id          BIGINT PRIMARY KEY AUTO_INCREMENT,
    activity_id BIGINT   NOT NULL,
    user_id     BIGINT   NOT NULL,
    signed_at   DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_activity_user (activity_id, user_id)
);
```

## 五、接口设计

### 5.1 统一响应格式

```json
{
  "code": 200,
  "msg": "success",
  "data": { ... }
}
```

| code | 含义 |
|------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未登录或 Token 失效 |
| 403 | 权限不足 |
| 500 | 服务器内部错误 |

### 5.2 主要接口列表

#### 认证

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/auth/login` | 登录，返回 JWT |
| POST | `/api/auth/register` | 注册 |
| POST | `/api/auth/logout` | 登出（前端清除 Token）|

#### 通知公告

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/notices` | 获取公告列表（分页，可按 category 筛选）|
| GET | `/api/notices/{id}` | 获取公告详情 |
| POST | `/api/notices` | 发布公告（需教师/管理员权限）|
| PUT | `/api/notices/{id}` | 修改公告 |
| DELETE | `/api/notices/{id}` | 删除公告 |

#### 校园活动

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/activities` | 获取活动列表 |
| GET | `/api/activities/{id}` | 获取活动详情 |
| POST | `/api/activities` | 发布活动（教师/管理员）|
| POST | `/api/activities/{id}/signup` | 报名参加活动 |
| DELETE | `/api/activities/{id}/signup` | 取消报名 |

## 七、运行说明

### 7.1 环境要求

- JDK 17+
- Node.js 18+
- MySQL 8.x
- Maven 3.6+

### 7.2 本地启动

```bash
# 1. 创建数据库
mysql -u root -p
CREATE DATABASE campus_db DEFAULT CHARSET utf8mb4;

# 2. 启动后端
cd backend
mvn spring-boot:run

# 3. 启动前端
cd frontend
npm install
npm run dev
```
